Download recording — Movesense / HeartBalance

A browser cannot download from a HeartBalance-firmware sensor — this is not a bug. That firmware's data service (FDF3) speaks the closed Movesense Whiteboard protocol: a versioned handshake + framing that only Movesense's libmds binary performs, so the sensor ignores every command a browser can send. Confirmed from the firmware, the Movesense SDKs, and HeartBalance's own apps. To get your recording: use the native iOS downloader (or the official app that exported it), then analyse it in HRV Studio. This page remains only as a raw-frame probe — useful for capturing bytes for reverse-engineering. Chrome/Edge only.

Nothing leaves your browser. Per its firmware, the sensor sends its stored recording as a JSON stream; this tool concatenates the frame payloads and lets you save them, then analyse the JSON. If it doesn't complete, the raw log shows the exact bytes — send it back (or an Android Bluetooth HCI snoop log of the official app downloading) and the exact protocol can be implemented.
